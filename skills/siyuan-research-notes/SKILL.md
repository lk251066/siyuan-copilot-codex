# SiYuan 调研笔记规则

## 适用场景
- 用户要求“调研/对比/找资料/给结论并附来源”。
- 需要把外部信息沉淀为可复现的思源笔记内容。

## 执行规则
- 结论至少使用两个来源交叉验证，优先官方文档和一手资料。
- 每条结论必须保留来源标题、原始 URL、访问日期、可信度。
- 不确定结论必须标记“待验证”，并给下一步验证命令或步骤。
- 不只贴链接，必须写“怎么复现”：命令、参数、输入输出示例。
- 涉及仓库必须写：仓库地址、分支、提交 SHA、关键文件路径。
- 涉及下载文件必须写：保存路径、文件名、版本、校验值（如 SHA256）。

## 输出格式
- 背景
- 结论
- 证据（来源 + 链接 + 日期 + 可信度）
- 复现步骤
- 验证结果（通过/失败 + 关键输出）
- 后续行动项（owner、截止时间、状态）

## 最小验证
- 回读一次目标块，确认已包含“来源链接 + 复现步骤 + 验证结果”三要素。
