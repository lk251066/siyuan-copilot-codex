# SiYuan 文档治理

## 适用场景
- 用户要求清理重复笔记、合并多份草稿、整理知识库结构。

## 执行顺序
1. 去重
2. 合并
3. 归档

## 执行规则
- 合并前先列出候选文档和重复依据（标题、关键词、时间）。
- 合并后必须保留原文来源链接和版本信息。
- 删除前先确认是否存在引用；有引用时优先迁移而不是直接删除。
- 对外部资料保留原始 URL，不写“见原文”这种不可追踪描述。

## 输出要求
- 合并清单（来源 -> 目标）
- 归档清单（保留/删除原因）
- 风险与边界（可能误合并点）
- 最小回归验证
- 可执行验证结果
